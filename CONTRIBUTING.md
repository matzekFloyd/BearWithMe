# Contributing

Thanks for your interest in contributing to BearWithMe.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies with `npm install`
4. Copy `.env.example` to `.env` and set local values
5. Start the app with `npm start`

## Branches and Pull Requests

- Create a feature branch from `main`
- Keep PRs focused and small
- Use clear commit messages
- Open a pull request with:
  - what changed
  - why it changed
  - how it was tested

## Code Style

- Follow existing project style
- Run tests before opening a PR: `npm test`
- Ensure the project builds: `npm run build`

## Reporting Bugs and Proposing Features

- Use GitHub issues for bugs and feature requests
- Include reproduction steps for bugs
- Include expected vs actual behavior

## Security

If you discover a security issue, follow `SECURITY.md` and report it privately.
